# PR Performance-Benchmark GitHub Action — Plan (v3)

**Status:** design locked + hardened against max-effort review; not yet implemented
**Date:** 2026-06-17 (v3 supersedes v2 same day)
**Owner:** Tim Treis

**v3 architecture change (supersedes v2):** the change-detection **mapper is dropped**. The action
benchmarks **all** public `get_*` measurement functions at the exposed-function level, **head vs
the current `main`**, and the speedup table itself shows what changed (~1.0× = untouched). This
deletes the entire static-analysis surface (former Component 2) and the merge-base/stacked-PR
logic. Benchmark cost is controlled by the matrix size / per-function budget, not by pre-selection.

## Goal

A GitHub Action that, when a maintainer labels a PR, benchmarks **every public `get_*` measurement
function** at PR-head vs **current `main`**, on deterministically generated **realistic** synthetic
cell images (cell-like label mask + channel images built from many Gaussian splats of varied size —
large, tiny — so area, intensity AND texture features all have real signal), across an
(image_size × object_count × reps) matrix (default 3×4×3), and posts the before/after/speedup table
as a sticky PR comment. Trigger via a label that is auto-removed when done.

## Grounded API facts (from `src/cp_measure/bulk.py`, verified)

- `_CORE` (8 fns): `radial_distribution, radial_zernikes, intensity, sizeshape, zernike, feret,
  texture, granularity` → **1 channel**. First positional arg is named `labels`
  (radial_*), `mask` (granularity), or `masks` (rest) — **call positionally**, never by keyword.
  `zernike`/`feret` ignore `pixels` (pure shape).
- `_CORRELATION` (4 fns): `costes, pearson, manders_fold, rwc` →
  signature `(pixels_1, pixels_2, masks)` — **2 channels, masks LAST** (core has masks FIRST).
- Legacy variants exist (`get_core_measurements(legacy=True)`, intensity #73, radial) and are
  **shipped** — must be in the entry-point table (D12).
- `utils.py` is imported by 5 modules (sizeshape, granularity, radial-dist/zernikes, 2×multimask)
  → file-closure mapping is hopelessly coarse (D2).
- bulk.py uses **lazy** imports for numba (`_numba_registries`) and multimask
  (`get_multimask_measurements`) → a static top-level import scan misses them (D3).
- Multimask fns take **two label masks** (`masks1, masks2`), no pixels — out of scope v1, but
  must be a distinct "skipped-unsupported" state (D20), not silent empty.
- `examples.py` has crude random/geometric generators — the new generator is the realistic
  replacement; do not extend the crude one.

## Components

### 1. Generator — `cp_measure/synth.py` (shipped + tested)
```python
generate(image_size, n_objects, n_channels=2, seed=0)
    -> (labels: int32[H,W], channels: float32[n_channels,H,W])
```
- **Label mask:** blue-noise (Poisson-disk) seeds with a min-spacing **bounded retry budget**
  → on failure to place `n_objects`, **raise loudly** (never silently under-place; D10) →
  watershed over a Gaussian-filtered random field (organic, non-convex boundaries) →
  **separation by grow-stop** (regions stop `k` px short of collision) instead of global
  erosion, so small cells survive (D11) → **min-area floor**: any seed whose realized cell
  < floor is dropped *and replaced* so realized count == requested; assert it (D10/D11).
  Per-cell sizes log-normal with the lower tail clamped to the floor → big + tiny + a couple of
  seed clusters, no degenerate 1px objects.
- **Channels (always 2, D22):** background + Σ multi-scale 2D Gaussian splats (large gradients /
  mid blobs / tiny puncta + clustered bursts), modulated by the mask envelope (object-correlated,
  some outside) + low photon/Gaussian noise → real intensity + Haralick-texture signal.
  ch0 and ch1 share the mask envelope but use **independent splat realizations**, tuned so
  pairwise correlation lands in a controlled band (target r ∈ [0.3, 0.7]).
- **dtypes:** `labels` int32, `channels` float32 (matches real microscopy; halves memory; D-mem).
- **Determinism:** output is a pure function of `(image_size, n_objects, n_channels, seed)` **for a
  fixed scipy/skimage build**. The bench action pins that build (Component 3); the unit suite
  pins it too. `generate()` is public API → **version-stamped** (`synth.__version__`), changes
  are semver-gated (D23).
- **Acceptance test (replaces visual sign-off, D24):** parametrised over the matrix **corners**
  (min-size×max-count, max-size×min-count): assert `realized_count == requested`, degenerate-
  object fraction == 0, eccentricity & area spread above floors, texture entropy above floor,
  ch0/ch1 correlation in band. Example PNGs saved as a CI artifact for optional human spot-check,
  but the **gate is the asserts**.

### 2. ~~Target mapper~~ — REMOVED in v3
No change-detection. The runner benchmarks **every** public `get_*` function (the `bulk`
core + correlation registries, incl. legacy variants where a `legacy` param exists), head vs
`main`. The speedup table is the "what changed" signal: a function the PR didn't touch lands at
~1.0× (within the noise floor) and is visually obvious. Cost is bounded by the matrix size and a
per-function timeout/budget (D7), not by pre-selection. This deletes the entire static-analysis
surface (AST symbol graph, git-diff parsing, the merge-base/stacked-PR logic) and its edge cases.

### 2 (was 3). Harness — fixture generation + runner + comparator
Benchmarks **all** public `get_*` functions (from `bulk.get_core_measurements()` +
`get_correlation_measurements()`, incl. legacy variants), **head vs `main`**.
- **Fixture provenance (D5/D17):** a dedicated **generation job** runs `synth.generate` from a
  **pinned ref** (a frozen tag of the generator + an exact-version env, independent of BASE and
  HEAD) — once per (size, count, seed) — and writes `.npy` to an **absolute temp dir outside both
  worktrees**, with a **sha256 checksum**. Uploaded as an artifact. Both measured runs only
  *load* these arrays; the measured code never calls the generator. This decouples generation
  from the code under test and from the bootstrapping problem (base needn't contain synth.py).
- **Two worktrees (D1):** `main` and HEAD, each `uv`-installed in its own env. Calls are
  **positional** per arity (D21): CORE `fn(masks, pixels=ch0)`, CORRELATION `fn(ch0, ch1, masks)`.
  Functions are enumerated from the **HEAD** registry (so a PR adding a feature still benchmarks
  it; a function absent on `main` is reported new, not compared).
- **One subprocess sweeps all functions per endpoint** (amortise import + JIT), with an
  untimed warm-up per function (numba JIT, D-warmup) then timed reps. Setup cost (2× uv install +
  cold JIT) is **counted against the budget** (D-setup).
- **Reps = distinct seeds (D14):** each rep uses a different generator seed → captures input
  variance, not just scheduler jitter. **Interleave** base/head per (cell, seed) to cancel slow
  runner drift. Report **median ratio** (primary) + min (secondary) + a **noise floor** estimated
  from repeated identical runs; deltas inside the noise floor are flagged "within noise."
- **speedup = base_time / head_time** (>1 = head faster); column header states the convention (D6).
- **Budget guards (D7):** per-call timeout (e.g. 120 s) and a total wall-clock budget; a cell that
  times out or would OOM is recorded as `timeout`/`skipped: budget` in the table — **logged, never
  silently dropped**. The expensive corner (2048²×max-count×texture@gray_levels=256) is the first
  to be guarded.
- **Thread pinning (D13):** set `OMP_NUM_THREADS, MKL_NUM_THREADS, OPENBLAS_NUM_THREADS,
  NUMEXPR_NUM_THREADS, VECLIB_MAXIMUM_THREADS, NUMBA_NUM_THREADS = 1`. Single-thread is
  representative **by design**: the repo forbids parallelism inside feature functions
  (no-parallelism-inside-functions rule), so this does not hide a threaded path that is supposed
  to exist.

### 4. Workflow — two-job split + comment (D8/D16)
- **Trigger:** `pull_request_target: types: [labeled]`; guard `github.event.action=='labeled' &&
  github.event.label.name=='benchmark'` AND **actor has write/admin** (`author_association ∈
  {OWNER,MEMBER,COLLABORATOR}` or an actor-permission API check) (D9). `workflow_dispatch`
  (pr_number) for manual + for testing the workflow itself (D18).
- **Job A — build/measure (unprivileged):** `permissions: {}` (no token, no secrets), runs the
  generation + both worktree benchmarks (untrusted fork code runs **here, with nothing to steal**),
  `actions/checkout` pinned by SHA with `persist-credentials: false`. Emits a results JSON
  artifact. Fork PRs reach Job A only via a GitHub **Environment with a required maintainer
  reviewer**.
- **Job B — report (privileged):** `needs: A`, `permissions: pull-requests: write`, **does NOT
  check out PR code** — downloads the artifact, renders the sticky comment
  (`<!-- cp-bench -->` marker, create-on-first-run else update; handle the no-existing-comment
  path and dedupe races, D-sticky), stamps generator-version + input checksum + matrix params
  (D23), and removes the label. Because Job B is separate and not the one the concurrency group
  cancels, label cleanup is reliable (D16).
- **Concurrency:** `group: pr-bench-${{ github.event.pull_request.number }}`,
  `cancel-in-progress: true` (D16).

## Matrix insight (intent preserved, vocabulary fixed — D15)
Object **count** is the knob and is **fixed across image sizes**; levels are chosen feasible at
the smallest size (512²) and reused at all sizes. This cleanly separates the two cost terms:
- **image_size ↑ @ fixed object count** → isolates **area-scaling** cost (the Zernike-PR win).
- **object_count ↑ @ fixed size** → isolates **per-object** cost.
Axis labelled `object count {N1<N2<N3<N4}` everywhere (never "sparse→dense").

## Timing-only caveat (D3-decision honoured, risk surfaced)
Decision 3 stays "timing only." But a fast-but-wrong PR would render as a green speedup, and this
repo's perf work hinges on bit-exactness. Mitigation kept cheap and non-gating: Job A also records
a **sha256 of each function's output** at base vs head on the shared inputs and the comment shows
a ✅/⚠️ "outputs identical / DIVERGED" flag per function. Correctness still lives in the test suite;
this is only a loud hint, not a gate.
